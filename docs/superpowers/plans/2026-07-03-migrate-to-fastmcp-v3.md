# Migrate Play Store MCP Server to `fastmcp` v3 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Migrate the server from the official MCP SDK's `mcp.server.fastmcp.FastMCP` to the standalone `fastmcp` package (v3.1+), preserving all current behavior exactly, so a later plan can add the `CODE_MODE` transform (which lives only in `fastmcp`).

**Architecture:** This is a framework swap, not a feature. `fastmcp` v3 is API-compatible with the SDK's FastMCP for the parts we use (`@mcp.tool()`, `@mcp.custom_route`, `mcp.run(transport=...)`), with three behavior-relevant differences we must adapt: (1) `FastMCP()` no longer accepts `host`/`port`/`transport_security` — those move to `run()` / the ASGI app; (2) reading HTTP headers inside a tool uses `fastmcp.server.dependencies.get_http_headers()` instead of `mcp.get_context().request_context`; (3) DNS-rebinding protection has no constructor equivalent, so we re-implement it with Starlette `TrustedHostMiddleware` on the HTTP app. The shared fallback client moves from `server._shared_state` / `lifespan_context` to a module-level `_shared_state` dict, decoupling us from framework internals.

**Tech Stack:** Python ≥3.11, `fastmcp` ≥3.1 (pulls its own `mcp`), `uv`/`uv.lock`, `pytest` + `pytest-cov` (branch), `ruff`, `mypy`, `bandit`, Starlette/uvicorn (transitive via fastmcp), `structlog`, `google-api-python-client`.

## Global Constraints

- **Python floor:** `requires-python = ">=3.11"` — unchanged.
- **Behavior parity (non-negotiable):** every one of the 117 tools, the `/health` and `/credentials` custom routes, per-request header credentials (`X-Google-Credentials` / `X-Google-Credentials-Base64`), the `PLAY_STORE_MCP_ADMIN_TOKEN` bearer auth on `/credentials`, `--read-only` / `PLAY_STORE_MCP_READ_ONLY`, and DNS-rebinding protection (`PLAY_STORE_MCP_DISABLE_DNS_REBINDING`) must behave identically after migration.
- **Transports:** `stdio` (default), `sse`, `streamable-http` must all still start via the same `--transport` / `MCP_TRANSPORT` CLI + env, on `--host`/`MCP_HOST` (default `127.0.0.1`) and `--port`/`MCP_PORT` (default `8000`).
- **Coverage:** keep **100 % branch coverage** (`pytest --cov=src/play_store_mcp --cov-branch`); the CI gate enforces it.
- **Clean gates before any push:** `ruff check src/ tests/`, `ruff format --check src/ tests/`, `uv run mypy src/`, `uv run --with bandit bandit -r src/`, `uv run --with pip-audit pip-audit` — all clean, mirroring the GitHub workflows.
- **No `CODE_MODE` in this plan.** Code-mode is Plan 2. This plan only enables the framework it depends on.
- **Commits/PRs carry no Claude attribution** (no author/footer/trailer).
- **`@mcp.tool()` (called form) is kept as-is** — fastmcp v3 supports it and returns the original function unchanged, so the ~94 tests that call tool functions directly keep working. Do **not** mass-rewrite decorators.

```python
def get_client_from_context() -> PlayStoreClient:
    """Resolve a PlayStoreClient for the current request.

    Per-request credentials in the X-Google-Credentials /
    X-Google-Credentials-Base64 headers take precedence; otherwise the shared
    client from the lifespan is used.

    Raises:
        PlayStoreClientError: if credentials are invalid or unavailable.
    """
    headers = get_http_headers() or {}

    if "x-google-credentials" in headers:
        try:
            creds_json = json.loads(headers["x-google-credentials"])
        except json.JSONDecodeError as e:
            raise PlayStoreClientError(f"Invalid JSON in X-Google-Credentials header: {e}") from e
        return PlayStoreClient(credentials_json=creds_json)

    if "x-google-credentials-base64" in headers:
        try:
            creds_bytes = base64.b64decode(headers["x-google-credentials-base64"])
            creds_json = json.loads(creds_bytes.decode("utf-8"))
        except (binascii.Error, UnicodeDecodeError, json.JSONDecodeError) as e:
            raise PlayStoreClientError(
                f"Invalid base64 or JSON in X-Google-Credentials-Base64 header: {e}"
            ) from e
        return PlayStoreClient(credentials_json=creds_json)

    client: PlayStoreClient | None = _shared_state.get("client")
    if client is not None:
        return client

    raise PlayStoreClientError(
        "No credentials provided. Set X-Google-Credentials or X-Google-Credentials-Base64 header, "
        "or configure server with GOOGLE_PLAY_STORE_CREDENTIALS environment variable."
    )
```

`get_http_headers()` returns `{}` under stdio (no HTTP request), so stdio falls through to the shared client — identical to today.

**Background:** `fastmcp` v3 dropped the SDK's `transport_security=` constructor arg. DNS-rebinding protection *is* Host-header validation, which Starlette's `TrustedHostMiddleware` provides. `fastmcp` attaches Starlette middleware through `mcp.http_app(middleware=[...])`, which returns an ASGI app served by `uvicorn`.

```python
def _dns_rebinding_disabled() -> bool:
    return bool(os.environ.get("PLAY_STORE_MCP_DISABLE_DNS_REBINDING"))


def _run_http(transport: str, host: str, port: int) -> None:
    """Serve a network transport with DNS-rebinding (Host-header) protection.

    fastmcp v3 has no constructor-level transport security; we attach Starlette
    TrustedHostMiddleware (which is exactly Host-header validation) unless
    PLAY_STORE_MCP_DISABLE_DNS_REBINDING is set.
    """
    middleware = []
    if not _dns_rebinding_disabled():
        allowed = [host, "localhost", "127.0.0.1", "[::1]"]
        middleware.append(Middleware(TrustedHostMiddleware, allowed_hosts=allowed))
    app = mcp.http_app(transport=transport, middleware=middleware)
    uvicorn.run(app, host=host, port=port)
```
